# Travel Agent 3.0

## Введение

Travel-agent-bot — это бот для помощи в планировании путешествий. Он помогает пользователю с выбором локации, предоставляет информацию о погоде, локациях, строит маршрут, позволяет делить расходы и ставить себе цели.

## Инструкция по запуску
1. Склонируйте репозиторий:
``git clone https://github.com/HghaVlad/Travel-agent-bot``
2. Перейдите в папку проекта:
``cd Travel-agent-bot``

3. Запустите Docker Compose:
```docker-compose up```

## Описание функционала
После запуска и входа в бота, пользователи могут выполнять следующие действия:

* **Создание путешествие:** Пользователи могут создавать путешествия, указывая интересующие локации и даты их посещения. Бот помогает организовать маршрут и предоставляет информацию о каждой локации.

* **Добавление друзей:** Пользователи могут добавлять других пользователей в друзья, чтобы легко делиться планами путешествий и расходами. Это также упрощает процесс поиска попутчиков среди уже знакомых людей.

* **Прогноз погоды:** Бот предоставляет детальный прогноз погоды на следующие 15 дней для выбранных локаций. Это позволяет пользователям лучше подготовиться к путешествию и выбрать подходящее время для посещения определенных мест.

* **Управление расходами:** Пользователи могут вносить расходы, связанные с путешествием, и разделять их с друзьями, аналогично функционалу Splitwise. Это упрощает учет расходов и управление долгами между участниками поездки.
* * При добавлении нового расхода, пользователь указывает его название и сумму. Затем можно выбрать, кто из участников путешествия должен будет внести свою долю.
* * Бот также позволяет просматривать все долги и расходы, а также отмечать расходы как погашенные.

* **Цели:** Пользователи могут ставить цели для своих путешествий, например, посетить определенные места или выполнить определенные задачи. Бот помогает отслеживать выполнение этих целей.
Пользователи могут добавлять новые цели, удалять их и отмечать выполненные. Это помогает лучше организовать путешествие и убедиться, что все запланированное выполнено.

* **Заметки:** Пользователи могут создавать текстовые заметки, добавлять фотографии или документы, которые будут связаны с конкретными путешествиями или локациями. Существует возможность редактировать, удалять и делиться заметками. Также пользователи могут управлять доступностью заметок, делая их видимыми для всех участников путешествия или только для себя.

* **Поиск попутчика:** Бот предлагает уникальную функцию поиска попутчиков, позволяя пользователям находить людей с похожими интересами и планами на путешествие. Это отличная возможность сделать свое путешествие более интересным и экономичным, разделив расходы и впечатления с новыми друзьями

## Описание внешних интеграций

* **OpenStreetMap (OSM):** Используется для определения точных координат интересующих локаций и планирования маршрутов между ними. Это открытая и масштабируемая географическая база данных, которая позволяет боту предоставлять пользователям актуальную и точную информацию о местоположении, делая процесс планирования путешествий более удобным и эффективным.

* **Visual Crossing Weather API:** Для предоставления детализированных прогнозов погоды на 15 дней вперед для любой точки мира. Благодаря доступу к этой информации через бесплатный функционал API, бот может информировать пользователей о погодных условиях в планируемых к посещению местах, помогая тем самым в выборе оптимального времени для путешествия.

* **Scikit-learn (sklearn.cluster.KMeans):** Эта библиотека машинного обучения используется для реализации алгоритма KMeans, который применяется для группировки пользователей на основе их предпочтений и планируемых путешествий. Это позволяет боту предлагать пользователям рекомендации попутчиков с похожими интересами, облегчая процесс поиска идеального компаньона для совместных поездок.


## Обоснованность выбора технологий и внешних интеграций
* PostgreSQL использовалась как основная база данных из-за ее мощности и надежности. Она отлично подходит для хранения сложных данных и обработки больших объемов информации, что важно для бота, управляющего данными о путешествиях, пользователях и их взаимодействиях.

* Aiogram выбран для разработки бота благодаря своей асинхронности и поддержке современных возможностей Telegram. Это делает бота быстрым и способным эффективно обрабатывать запросы пользователей.

* Redis используется для кэширования данных и сессий, чтобы ускорить отклик бота и снизить нагрузку на основную базу данных. Это улучшает производительность, особенно при работе с внешними API, предотвращая необходимость повторных запросов за короткий промежуток времени.
